# 素材库精品原片库设计

日期: 2026-04-29
状态: 已确认方向, 待实现

## 背景

清华哥确认素材库的核心目标不是管理电脑里的所有文件, 而是服务未来短视频剪辑:

- 素材主要是清华哥实际拍摄的照片和视频原片.
- 剪辑时命中文案语义后, 优先从本地素材库找真实画面, 找不到再 AI 生成.
- 当前先用 `~/Downloads/` 作为演示数据源跑通能力, 因为里面已有大量历史短视频素材.
- 未来可以切到桌面专用文件夹, 例如 `~/Desktop/清华哥素材库/`, 只放 50-200 个精品素材.

## 产品定位

素材库从“文件浏览器”改成“精品原片货架”:

1. 找得快: 按业务场景和标签找, 不是按杂乱下载目录找.
2. 看得懂: 每条素材展示画面摘要、镜头类型、横竖屏、质量和适合用途.
3. 能整理: 新素材先进待整理, AI 给建议分类和标签, 清华哥只做确认.
4. 能调用: 做视频、数字人剪辑、生图参考时能用文案语义检索素材.

## 默认大类

正式素材库默认按 8 个大类组织:

| 目录 | 用途 |
|---|---|
| `00 待整理` | 新丢进来的原始素材, 等 AI 识别和人工确认 |
| `01 演讲舞台` | 上台、观众、掌声、会场、合影、舞台空镜 |
| `02 上课教学` | 课堂、板书、课件、学员互动、讲课片段 |
| `03 研发产品` | 研发现场、电脑、工具、产品演示、团队讨论 |
| `04 出差商务` | 机场、高铁、酒店、城市、客户现场、商务会面 |
| `05 做课素材` | 录课、课程封面、PPT、讲义截图、课程花絮 |
| `06 空镜补画面` | 办公室、手部、键盘、书、咖啡、街景、转场镜头 |
| `07 品牌资产` | logo、海报、封面、固定视觉物料 |

当前 `~/Downloads/` 演示阶段不要求真实移动文件, 先做虚拟归类和筛选展示.
未来切到 `~/Desktop/清华哥素材库/` 后, `approve` 可以升级为真实移动文件.

## 识别与打标

当前 D-087 的 `materials.tag` 只看文件名、路径和元数据, 不足以识别真实画面. 新版需要分两级:

1. 快速层: 文件名、路径、后缀、尺寸、时长、横竖屏、缩略图是否存在.
2. 画面层: 图片直接识别; 视频抽 3-5 张关键帧后识别.

每条素材输出结构化画像:

- `category`: 上面的 8 个大类之一.
- `visual_summary`: 一句话画面摘要.
- `tags`: 5-12 个中文标签.
- `scene`: 演讲 / 上课 / 研发 / 出差 / 空镜 / 品牌等.
- `shot_type`: 人物口播 / 观众反应 / 环境空镜 / 手部特写 / 屏幕录制 / 合影等.
- `orientation`: 竖屏 / 横屏 / 方图.
- `quality_score`: 0-100, 只把可用素材推到前面.
- `usage_hint`: 适合开头 / 转场 / 证明 / 情绪 / 背景 / 结尾.
- `relevance_score`: 是否贴近清华哥业务.

DeepSeek 不能稳定做视觉识别时, 视觉层走 Opus/OpenClaw 视觉能力; 不可用时降级为快速层, 但 UI 必须标明“未识别画面”.

## 展示方式

首页不要优先展示 Downloads 原始文件夹. 首页应展示:

- 8 个业务大类卡片, 每类显示素材数、可用素材数、最近新增.
- “待整理”入口, 用来批量确认 AI 建议.
- 全库搜索, 可搜画面摘要、标签、文件名、分类.
- “当前演示源: Downloads”提示, 让用户知道这是临时数据源.

进入大类后:

- 默认按质量和最近新增排序.
- 图片和视频统一卡片, 视频显示时长和横竖屏.
- 右侧预览展示画面摘要、标签、用途建议、使用次数.
- 大预览页支持“用它做视频”, 并写入 usage log.

## 剪辑调用

后续生产链路需要新增素材检索接口:

输入:

- 文案段落或镜头描述.
- 期望素材类型: 图片 / 视频 / 空镜 / 证明画面.
- 画幅: 竖屏优先 / 横屏可用.

输出:

- 匹配素材列表, 按 `category + tags + visual_summary + usage_hint + quality_score` 排序.
- 每条返回本地文件路径、缩略图、推荐理由.

原则:

- 本地素材命中优先于 AI 生成.
- 低质量、无缩略图、识别失败的素材不自动进剪辑, 只允许人工选择.

## 实施分期

### 第一阶段: 跑通精品原片库 MVP

- 保留 `~/Downloads/` 作为默认演示源.
- 增加 8 个业务大类和虚拟归类.
- 首页从“文件夹大卡片”改成“业务大类大卡片”.
- 补 `settings.materials_root` 默认值和设置可保存能力.
- 打标输出结构化画像, 即使暂时没有视觉识别也要保留字段.
- 新增业务分类/搜索相关测试.

### 第二阶段: 视觉识别

- 图片走视觉识别.
- 视频抽关键帧后识别.
- UI 标明识别来源: 画面识别 / 文件名推断 / 未识别.
- 限量批处理, 避免一次扫完整个 Downloads 烧太多 credits.

### 第三阶段: 剪辑检索

- 新增 `/api/material-lib/match` 语义检索.
- 做视频/数字人剪辑优先调用本地素材.
- 没命中再调用生图/生视频.

## 验收标准

- 用 Downloads 可跑通一次扫描、分类、搜索、预览、打标.
- 页面首页展示业务大类, 不再让用户面对杂乱 Downloads 文件夹.
- 能明确看出哪些素材适合演讲、上课、研发、出差、空镜、做课.
- 未来切到桌面专用文件夹只改设置, 不改业务代码.
- QA 必须做真实浏览器闭环: 截图、console clean、点击分类/搜索/预览/待整理.
