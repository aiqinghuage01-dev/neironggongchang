## 一、产品概述

### 1.1 产品名称

**清华哥内容工厂**（Content Factory）

### 1.2 一句话定义

一个运行在本地电脑上的 Web 控制台（打开 `localhost:3000` 即用），将清华哥现有的 32 个 AI 技能、28 条知识库、声音克隆、数字人、剪辑、发布等能力串成一条**可视化内容流水线**，实现从素材采集到内容发布的一站式操作。

### 1.3 核心问题

| 现状痛点 | 目标状态 |
|---------|---------|
| 内容生产散落在 10+ 个工具中（飞书、Hermes、石榴数字人、剪映、公众号后台、小红书…） | 打开一个网址，所有操作在一个界面完成 |
| 每次生产内容需要手动在工具间跳转、复制粘贴 | 内容像流水线产品一样自动流转，点一下进入下一步 |
| 没有统一的素材库、选题库、成品库 | 所有素材/文案/视频/数据集中管理、可搜索、可复用 |
| 知识库、技能、模板散落在文件系统各处 | 技能面板化，知识库可视化，模板一键调用 |
| 无法直观展示"我的内容生产系统"给学员看 | 打开工厂界面就是最好的展示——"这就是我的系统" |

### 1.4 目标用户

**仅清华哥本人**。不考虑多用户、不考虑学员使用、不考虑部署到服务器。纯本地单机使用。

### 1.5 成功标准

- 日常内容生产全流程（从选题到发布）可在内容工厂内完成，不需要跳出去
- 单条内容从素材到成品的时间缩短 50%+
- 拍短视频时打开界面给学员看，学员觉得"这东西真棒，很想要"

## 二、用户画像与使用场景

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| 身份 | 陈清华（清华哥），新媒体知识付费从业者 |
| 业务 | 实体老板AI获客训练营 |
| 技术水平 | 非技术背景，但高度适应AI工具，日常使用Hermes、飞书、Claude |

| 内容产出量 | 日均 3-5 条短视频文案 + 不定期公众号文章 + 投流素材 |

### 2.2 核心使用场景

#### 场景1：日常短视频内容生产（最高频）

```
早上9:00 — 打开 localhost:3000
→ 看到今日热点推荐 + 待处理素材队列
→ 选一个热点，点"生成文案"，选模板（热点爆款/真诚版/投流版）
→ 文案生成后预览、微调
→ 点"生成语音"，用克隆声音朗读文案
→ 点"生成视频"，数字人口播
→ 点"发布"，选平台（抖音/视频号/小红书）
→ 发布后在数据看板跟踪效果
```

#### 场景2：批量内容生产

```
→ 进入"批量模式"
→ 输入5个选题 / 导入选题库
→ 选模板和风格
→ 一键批量生成5条文案
→ 逐条确认或微调
→ 批量生成语音和视频
→ 设定发布时间，定时发布
```

#### 场景3：公众号文章深度创作

```
→ 在素材区整理参考资料
→ 选择"公众号文章"模板
→ AI基于知识库+参考资料生成长文
→ 预览排版效果
→ 一键发布到公众号
```

#### 场景4：录音/口述转内容

```
→ 上传一段录音文件
→ 自动转写文字
→ 选择改写风格（保留口吻/爆款改写/投流风格）
→ 生成成品文案
→ 后续走视频生成/发布流程
```

#### 场景5：素材采集与选题管理

```
→ 搜索小红书/抖音热点
→ 采集同行爆款文案
→ 素材自动入库，打标签
→ 选题库按热度/时效排序
→ 从选题库一键发起内容生产
```

#### 场景6：给学员展示

```
→ 拍短视频时打开工厂界面
→ 展示完整的内容生产流水线
→ 展示看板上的内容卡片在各工位之间流转
→ 学员看到系统化的生产方式，产生"我也想要"的欲望
```

### 3.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端框架** | Next.js 14 (App Router) + React 18 | Mac上已有Node 22，零额外安装；SSR+CSR混合渲染；API Routes省掉独立后端 |
| **UI组件库** | Tailwind CSS + shadcn/ui | 开发快、界面现代、深色主题开箱即用 |
| **数据库** | SQLite (via better-sqlite3) | 纯本地文件，无需装数据库，够用 |
| **AI引擎** | Claude Opus 代理 (localhost:3456) | 已有，直接调用，写文案/改写全靠它 |
| **文件存储** | 本地文件系统 | 视频、音频、素材全存本地，直接读写 |
| **技能调用** | Shell exec → 各 Skill 的 CLI/脚本 | 复用现有32个skill，不重新开发 |
| **飞书集成** | lark-cli | 已安装，文档/表格/消息全通 |
| **视频处理** | FFmpeg | macOS自带或brew安装，剪辑/转码/字幕 |
| **实时通信** | Server-Sent Events (SSE) | AI生成文案时实时流式输出到前端 |

## 四、功能模块详细设计

### 4.1 模块总览

| 模块 | 优先级 | 描述 |
|------|--------|------|
| **首页看板** | P0 | 内容卡片看板，可视化流水线 |
| **素材中心** | P0 | 热点采集、爆款提取、录音转写、选题库 |
| **文案引擎** | P0 | 模板选择、知识库注入、AI生成、人工微调 |
| **视频工坊** | P1 | 语音合成、数字人视频、智能剪辑 |
| **发布中心** | P1 | 多平台发布、定时发布 |
| **数据看板** | P2 | 各平台数据回收、效果分析 |
| **知识库管理** | P1 | 可视化浏览/编辑知识库 |
| **资产库** | P2 | 历史文案、视频、素材的统一管理 |

### 4.1.1 核心设计理念：技能隐身，动作外露

**原则**：用户在界面上看到的永远是业务动作（"改写""变朋友圈""转投流版""生成口播视频"），而不是技术概念（"调用热点爆款文案skill"）。32个Skill作为底层引擎驱动各环节，但用户无需知道它们的存在。

**实现方式**：每个工位根据当前内容的阶段和类型，自动浮现可用的动作按钮。点击按钮 = 后台调用对应Skill + 传入当前内容 + 自动注入知识库。

**动作按钮映射示例**：

| 用户看到的按钮 | 实际调用的Skill | 出现在哪个工位 |
|--------------|---------------|--------------|
| "热点爆款改写" | 热点爆款文案 / 热点爆款文案2 / 热点爆款文案3 | 文案引擎 |
| "真诚风改写" | 热点文案真诚版 | 文案引擎 |
| "转投流版" | touliu-agent / strong-marketing-ad-copy | 文案引擎（文案完成后的二次加工） |
| "变朋友圈" | 朋友圈金句知识库 + Claude Opus 改写 | 文案引擎（文案完成后的衍生） |
| "写公众号" | 公众号文章 | 文案引擎 |
| "录音转文案" | 录音文案改写 | 素材中心（录音上传后） |
| "生成口播视频" | shiliu-digital-human | 视频工坊 |
| "采集小红书" | xiaohongshu-feishu-bitable-sync | 素材中心 |
| "批量出选题" | piliang-xuanti-zhushou | 素材中心 |
| "发公众号" | 公众号文章（发布环节） | 发布中心 |
| "知识库体检" | 黄金五环评审（学员版） | 知识库管理 |

**一条内容卡片的动作流转示例**：

```
卡片状态：素材池
  可用动作：[热点爆款改写] [真诚风改写] [录音转文案] [行业改写]
                         点击任一按钮
                              |
卡片状态：文案生产中 → 文案完成
  可用动作：[转投流版] [变朋友圈] [写公众号长文] [微调编辑]
                         点击任一按钮
                              |
卡片状态：文案完成
  可用动作：[生成口播视频] [跳过视频直接发布]
                         点击任一按钮
                              |
卡片状态：视频完成 → 待发布
  可用动作：[发抖音] [发视频号] [发小红书] [发公众号] [定时发布]
```

**关键交互**：点击动作按钮后，右侧弹出配置面板（可选参数、知识库注入项等），确认后一键执行，执行过程流式展示，完成后卡片自动流转到下一阶段。

### 4.2 首页看板（P0）

#### 4.2.1 功能描述

首页是一个**看板视图**（类似 Trello/飞书项目），每条内容是一张卡片，卡片在不同阶段之间拖拽流转。

#### 4.2.2 看板列（Columns）

| 列名 | 含义 | 卡片状态 |
|------|------|---------|
| **素材池** | 已采集但未处理的素材（热点、录音、爆款链接） | raw |
| **选题确认** | 已确认要做的选题 | topic_confirmed |
| **文案生产中** | 正在AI生成/人工调整的文案 | copywriting |
| **文案完成** | 文案已定稿，待生成视频 | copy_done |
| **视频生产中** | 正在生成语音/数字人/剪辑 | video_producing |
| **待发布** | 视频已完成，等待发布 | ready_to_publish |
| **已发布** | 已发到各平台 | published |
| **归档** | 已完成全流程，含数据回收 | archived |

#### 4.2.3 卡片信息

每张卡片显示：
- **标题**：选题/素材标题
- **标签**：类型标签（热点/口播/投流/公众号）、风格标签（爆款/真诚/强营销）
- **缩略图**：如果有视频则显示封面
- **进度指示器**：当前所在阶段的进度条
- **时间**：创建时间、最后操作时间
- **快捷操作按钮**：根据当前阶段显示"下一步"按钮

#### 4.2.4 交互设计

- **拖拽**：卡片可在列之间拖拽以改变状态
- **点击卡片**：右侧弹出详情面板，显示该内容的完整信息和操作按钮
- **顶部工具栏**：
  - "新建内容" 按钮
  - 筛选器（按类型、标签、时间）
  - 搜索框
  - 视图切换（看板视图 / 列表视图 / 时间线视图）
- **统计栏**：今日生产数、本周发布数、待处理数

### 4.3 素材中心（P0）

#### 4.3.1 功能描述

统一的素材采集和管理入口，所有内容生产的原材料从这里进入系统。

#### 4.3.2 子模块

##### A. 热点采集

| 字段 | 描述 |
|------|------|
| **功能** | 一键抓取当前热点话题 |
| **数据来源** | 微博热搜、抖音热榜、百度热搜、小红书热门（通过 tavily-search 和爬虫） |
| **操作流程** | 点击"刷新热点" → 展示热点列表 → 点击"采集"将热点加入素材池 |
| **底层调用** | tavily skill / 小红书MCP |
| **输出** | 热点标题 + 简介 + 热度值 → 存入素材库 |

##### B. 同行爆款提取

| 字段 | 描述 |
|------|------|
| **功能** | 输入URL或关键词，提取同行爆款文案 |
| **支持平台** | 小红书、抖音、视频号、公众号 |
| **操作流程** | 粘贴URL / 输入关键词搜索 → 展示内容列表 → 选择采集 → 提取文案正文 |
| **底层调用** | xiaohongshu-feishu-bitable-sync / agent-browser / summarize skill |
| **输出** | 原文案 + 数据指标（点赞/评论/收藏）→ 存入素材库 |

##### C. 录音上传与转写

| 字段 | 描述 |
|------|------|
| **功能** | 上传录音文件，自动转写为文字 |
| **支持格式** | mp3, wav, m4a, aac, ogg |
| **操作流程** | 拖拽上传/选择文件 → 自动转写（调用Whisper或飞书妙记） → 展示转写结果 → 可直接发起文案改写 |
| **底层调用** | Whisper API / 飞书妙记 / 录音文案改写 skill |
| **输出** | 转写文本 → 存入素材库 |

##### D. 选题库

| 字段 | 描述 |
|------|------|
| **功能** | 管理所有选题，支持排序、筛选、打标签 |
| **数据来源** | 手动添加 + 热点采集 + 爆款提取 + 批量选题助手生成 |
| **字段** | 选题标题、来源、标签、热度评分、创建时间、状态（待用/已用/过期） |
| **操作** | 从选题一键创建内容卡片 → 自动进入看板流水线 |
| **底层调用** | piliang-xuanti-zhushou skill（批量生成选题） |

##### E. 飞书文档/妙记导入

| 字段 | 描述 |
|------|------|
| **功能** | 从飞书拉取已有文档、妙记、会议纪要作为素材 |
| **操作流程** | 搜索飞书文档 → 预览内容 → 导入为素材 |
| **底层调用** | lark-cli docs / lark-cli minutes |

#### 4.3.3 素材数据模型

```
Material {
  id: UUID
  title: String              // 素材标题
  type: Enum                 // hot_topic | competitor | recording | topic | feishu_doc | manual
  source: String             // 来源平台/URL
  content: Text              // 素材正文内容
  tags: String[]             // 标签
  heat_score: Int            // 热度评分(0-100)
  metrics: JSON              // 原始数据指标（点赞/评论等）
  file_path: String?         // 关联文件路径（录音/截图等）
  status: Enum               // unused | in_use | used | expired
  created_at: DateTime
  updated_at: DateTime
}
```

### 4.4 文案引擎（P0）

#### 4.4.1 功能描述

内容工厂的核心模块。将素材 + 知识库 + AI能力 + 模板体系结合，高效生成各类文案。

#### 4.4.2 文案生成流程

用户不需要"选模板"这个技术步骤。流程是动作驱动的：

```
Step 1: 在素材卡片上点击动作按钮（如"热点爆款改写""真诚风改写""转投流"）
         ↓
Step 2: 右侧弹出配置面板 — 可微调参数（风格、长度、业务植入程度）
         系统自动推荐知识库注入项（可手动增减）
         ↓
Step 3: 点"开始生成" → AI流式输出，实时预览
         ↓
Step 4: 人工微调（富文本编辑器）
         ↓
Step 5: 定稿 → 卡片自动流转到"文案完成"
         此时卡片上浮现新的动作按钮：[转投流版] [变朋友圈] [写公众号长文] [生成口播视频]
```

**文案二次加工**：定稿后的文案不是终点，而是新动作的起点。点"转投流版"会基于当前文案调用投流skill再生成一版；点"变朋友圈"会提取核心观点生成3-5条朋友圈文案。一条素材可以衍生出多条不同形态的内容。

#### 4.4.3 文案类型与对应模板

| 类型 | 可用模板（对应Skill） | 适用场景 |
|------|---------------------|---------|
| **短视频口播** | 热点爆款文案、热点爆款文案2、热点爆款文案3、热点文案真诚版、人物争议热点文案 | 日常短视频内容 |
| **投流文案** | touliu-agent、strong-marketing-ad-copy | 付费投放素材 |
| **行业改写** | 行业爆款改写、爆款改写（学员版） | 借鉴同行爆款 |
| **录音改写** | 录音文案改写 | 口述内容整理 |
| **公众号文章** | 公众号文章 | 深度长文 |
| **趋势热点** | trend-business-persona-rewriter | 趋势+业务+人设 |

#### 4.4.4 知识库注入机制

文案生成时，系统自动：
1. 根据选题内容，从28条知识库中匹配相关素材
2. 推荐3-5条最相关的知识库项（可手动增减）
3. 将选中的知识库内容注入AI提示词
4. 确保生成的文案使用真实案例和数据（不编造）

**可注入的知识库分类**：
- 业务定位（3条）：让文案准确反映业务定位
- 学员案例（4条）：注入真实案例增强说服力
- 方法论（12条）：体现专业深度
- 金句话术（3条）：增强文案感染力
- 行业洞察（4条）：体现行业认知
- 人设经历（2条）：强化人设可信度

#### 4.4.5 文案编辑器

- 富文本编辑器，支持 Markdown
- 左侧：AI生成结果（可重新生成/局部重写）
- 右侧：知识库参考面板（可拖拽内容到编辑器）
- 底部：文案质检评分（基于120分制质检评分卡，自动打分）
- 字数统计、预估口播时长

#### 4.4.6 文案数据模型

```
Copywriting {
  id: UUID
  content_id: UUID           // 关联的内容卡片ID
  material_id: UUID?         // 关联的素材ID
  type: Enum                 // short_video | ad | article | rewrite
  template: String           // 使用的模板/Skill名称
  prompt_config: JSON        // 生成参数配置
  knowledge_items: String[]  // 注入的知识库项ID列表
  draft_versions: JSON[]     // 历史版本（每次生成/修改都保留）
  final_content: Text        // 定稿内容
  quality_score: Int?        // 质检评分
  word_count: Int
  estimated_duration: Int    // 预估口播秒数
  status: Enum               // generating | draft | reviewing | finalized
  created_at: DateTime
  updated_at: DateTime
}
```

### 4.5 视频工坊（P1）

#### 4.5.1 功能描述

将定稿文案转化为可发布的视频内容。包含语音合成、数字人生成、智能剪辑三个子环节。

#### 4.5.2 语音合成

| 字段 | 描述 |
|------|------|
| **功能** | 用清华哥的克隆声音朗读文案 |
| **输入** | 定稿文案文本 |
| **配置项** | 语速（0.8x-1.5x）、停顿控制、情感风格 |
| **输出** | 音频文件（mp3/wav） |
| **底层方案** | 待确认：剪映TTS / HeyGen / 其他声音克隆API |
| **预览** | 生成后可在线播放试听，不满意可调整参数重新生成 |

#### 4.5.3 数字人视频生成

| 字段 | 描述 |
|------|------|
| **功能** | 音频 + 数字人形象 → 口播视频 |
| **输入** | 语音音频文件 |
| **配置项** | 数字人形象选择、背景选择、画面比例（9:16竖屏/16:9横屏） |
| **输出** | 口播视频文件（mp4） |
| **底层调用** | shiliu-digital-human skill / 石榴16AI API |
| **状态追踪** | 视频生成通常需要几分钟，显示进度条，完成后通知 |

#### 4.5.4 智能剪辑

| 字段 | 描述 |
|------|------|
| **功能** | 对口播视频做后期处理 |
| **支持的操作** | 自动加字幕（支持样式选择）、添加BGM（可选BGM库）、片头片尾模板、画面转场效果、视频裁剪/拼接 |
| **底层调用** | FFmpeg（字幕烧录/音频混合/转场/裁剪）、Remotion（动态字幕/动画效果，已有remotion-poster技能） |
| **预览** | 剪辑后可在线预览视频效果 |

#### 4.5.5 视频数据模型

```
Video {
  id: UUID
  content_id: UUID           // 关联的内容卡片ID
  copywriting_id: UUID       // 关联的文案ID
  audio_path: String?        // 语音文件路径
  audio_config: JSON         // 语音合成参数
  video_path: String?        // 原始数字人视频路径
  edited_video_path: String? // 剪辑后视频路径
  edit_config: JSON          // 剪辑参数（字幕样式/BGM/转场等）
  thumbnail_path: String?    // 封面图路径
  duration: Int              // 视频时长（秒）
  resolution: String         // 分辨率
  status: Enum               // audio_generating | audio_done | video_generating | video_done | editing | edit_done
  created_at: DateTime
  updated_at: DateTime
}
```

### 4.6 发布中心（P1）

#### 4.6.1 功能描述

将成品内容一键发布到各平台，支持定时发布和批量发布。

#### 4.6.2 支持的发布渠道

| 渠道 | 发布方式 | 内容类型 | 底层方案 |
|------|---------|---------|---------|
| **公众号** | API直接发布 | 图文文章 | 公众号文章 skill（已有完整流程） |
| **抖音** | 半自动（打开发布页+自动填充） | 短视频 | 浏览器自动化 / 抖音开放平台API |
| **视频号** | 半自动 | 短视频 | 浏览器自动化 |
| **小红书** | 半自动 | 图文/视频 | 小红书MCP / 浏览器自动化 |
| **飞书** | API直接发布 | 消息/文档 | lark-cli |

> **"半自动"说明**：部分平台不开放发布API，采用浏览器自动化方式——自动打开发布页面、填入标题/内容/封面，清华哥只需点"发布"确认。

#### 4.6.3 发布配置

- **标题编辑**：每个平台可单独调整标题（不同平台风格不同）
- **封面选择**：从视频中截取 / 上传自定义封面 / AI生成封面
- **话题标签**：根据内容自动推荐话题标签，可手动调整
- **定时发布**：选择发布时间，系统定时触发
- **批量发布**：勾选多条内容 → 选择平台 → 批量发布

#### 4.6.4 发布记录数据模型

```
Publication {
  id: UUID

  platform: Enum             // wechat_article | douyin | weixin_channel | xiaohongshu | feishu
  platform_post_id: String?  // 平台上的文章/视频ID
  platform_url: String?      // 平台上的链接
  title: String              // 发布标题
  cover_path: String?        // 封面图路径
  hashtags: String[]         // 话题标签
  scheduled_at: DateTime?    // 计划发布时间
  published_at: DateTime?    // 实际发布时间
  status: Enum               // draft | scheduled | publishing | published | failed
  error_message: String?     // 发布失败原因
  created_at: DateTime
}
```

### 4.9 技能配置（P2 — 管理视角，日常无需进入）

#### 4.9.1 设计理念

**日常使用时不需要"技能面板"**。所有技能已经融入各工位的动作按钮中（见 4.1.1）。

本模块仅用于**管理和配置**：新增技能、调整技能参数默认值、查看技能调用统计、排查问题。

#### 4.9.2 功能

- **技能列表**：展示所有已注册的Skill，显示名称、映射到哪些动作按钮、调用次数
- **动作映射编辑**：配置"用户看到的按钮名" ↔ "底层Skill" 的映射关系
- **默认参数**：为每个Skill设置默认参数（如知识库注入项、风格偏好），减少每次使用时的配置步骤
- **调用日志**：查看每次Skill调用的输入/输出/耗时，用于排查和优化

#### 4.9.3 底层实现

- 扫描 `~/Desktop/skills/` 目录，读取每个Skill的 `SKILL.md` 获取元信息
- 映射关系存储在 SQLite 的 `skill_action_mapping` 表中
- 通过Shell执行Skill命令，执行结果实时流式返回
