## 发现的问题

### P1 - 数据看板排行里的历史作品「看」按钮点不开

- 现象: 数据看板 TOP 行显示历史作品, 点击「看」后没有打开作品详情抽屉.
- 证据: `/tmp/_ui_shots/qa_works_20260429131851_08_analytics_top_old.png` 有 TOP 历史作品; `/tmp/_ui_shots/qa_works_20260429131851_09_analytics_after_open_click.png` 点击后仍停在看板, 无抽屉.
- 代码证据: `web/factory-works.jsx:188-190` 只在当前 `works` 列表里 `find` 对应 id; 当前列表受 `sinceFilter=today`、tab、搜索词影响, 看板数据可以来自列表之外.
- 影响: 老作品只要不在当前时间筛选/前 300 条中, 看板排行就只能看数字, 不能回看作品内容.

### P1 - 「留这版 / 删这版」写入成功但 UI 不更新

- 现象: 点击「留这版」后, API metadata 已写入 `user_action=kept`, 但按钮仍显示 `👍 留这版`, 没有 `✓`.
- 证据: `/tmp/_ui_shots/qa_works_20260429131851_05_after_keep_click.png`; 浏览器报告里 `keep_button_before` 与 `keep_button_after` 都是 `👍 留这版`, 但 `keep_api_metadata` 已包含 `user_action`.
- 代码证据: `web/factory-works.jsx:404-422` 依赖传入的 `work.metadata` 计算按钮状态; `web/factory-works.jsx:461` 只 dispatch `works-action-changed`, 页面没有对应 listener 重读作品.
- 影响: 老板点击后不知道是否生效, 再点可能仍按旧状态发送, 行为记忆标记容易误操作.

### P2 - 完播率输入框标「%」但后端只接受 0-1

- 现象: UI 字段写「完播率 %」, 输入 `80` 保存弹出 `completion_rate 最多 1`; 输入 `0.8` 才成功.
- 证据: `/tmp/_ui_shots/qa_works_20260429131851_06_metrics_invalid_percent.png`; curl `completion_rate=80` -> `422`, `completion_rate=0.8` -> `200`.
- 代码证据: `web/factory-works.jsx:721` label 是 `完播率 %`; `backend/api.py:1778` 约束是 `completion_rate <= 1`.
- 影响: 正常用户会按百分比填 `80`, 保存失败.

### P2 - 搜索只在前 300 条结果里过滤, 老作品可能搜不到

- 现象: 现场 `8000` 有 342 条作品; 第 301 条 `work_id=58`, query `cover_839366f3a4de`, `limit=300&q=...` 查不到, `limit=1000&q=...` 能查到.
- 证据:
  - `GET /api/works?limit=300&q=cover_839366f3a4de` -> `[]`
  - `GET /api/works?limit=1000&q=cover_839366f3a4de` -> `[58]`
- 代码证据: `backend/api.py:1642-1650` 先 `list_works(limit=limit)` 再在内存里做 `q` 过滤; 前端 `web/factory-works.jsx:46` 固定 `limit=300`.
- 影响: 作品越多, 越可能“明明有但搜不到”.

### P2 - 图片作品存在大量无预览/无下载的占位卡

- 现象: 图片 tab 出现只有图标占位的卡片; 详情页也只有占位, 没有「下载原图」.
- 证据: `/tmp/_ui_shots/qa_works_20260429131851_11_image_tab_placeholders.png`, `/tmp/_ui_shots/qa_works_20260429131851_12_image_drawer_no_download.png`
- 现场只读快照: `8000` 前 300 条里 `image_missing_preview=59`.
- 代码证据: `backend/api.py:1655-1671` 只有本地文件存在才返回 `local_url/thumb_url`; mmbiz-only 或缺失本地文件的图片会变成不可查看占位.
- 影响: 作品库里一批图片看不到图也下载不了, 老板会感知为“作品库坏了”.

## 复现步骤

1. 数据看板问题: 准备一条非今天作品并录入高播放指标; 打开作品库默认今天; 进「数据看板」; 点击 TOP 行「看」.
2. 留/删按钮问题: 打开任意作品详情; 点击「留这版」; 观察按钮没有 `✓`; 查 API 可见 metadata 已更新.
3. 完播率问题: 作品详情 -> 各平台数据 -> 录入 -> 完播率填 `80` -> 保存.
4. 搜索问题: 用现场数据搜索第 300 条之后的标题片段, 或 API 对比 `limit=300` 与 `limit=1000`.
5. 图片占位问题: 图片 tab 打开无 `thumb_url/local_url` 的图片作品.
